# Contributing

SheSafe is an early-stage safety mapping concept. Contributions should keep the project careful, privacy-preserving, and public-interest oriented.

## Useful Contributions

- Improve privacy, moderation, accessibility, or documentation.
- Fix reproducible bugs in the map, pin submission, or interface flow.
- Clarify product boundaries and safety language.
- Add tests or examples that use synthetic data only.

## Contribution Rules

- Do not add panic-oriented, sensational, or hostile language.
- Do not add invasive tracking, background location harvesting, or unnecessary identity collection.
- Do not commit secrets, private reports, precise personal location histories, or real sensitive safety data.
- Follow [ecosystem-standards](https://github.com/sparshsam/ecosystem-standards).

## Process

1. Open a focused issue or pull request.
2. Explain the user or community need.
3. Include testing notes where relevant.
4. Keep changes small enough to review carefully.
