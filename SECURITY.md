# Security Policy

SheSafe is an early-stage public-interest safety mapping concept. Security and privacy issues are treated seriously because the project may involve sensitive public-safety context.

## Reporting A Vulnerability

Please report security issues through GitHub by opening a private vulnerability report where available, or by contacting the maintainer directly.

Do not publish exploit details, credentials, API keys, personal location data, moderation bypasses, or sensitive safety reports in public issues.

## Sensitive Data Rules

- Do not commit `.env` files, API keys, tokens, or service credentials.
- Do not publish precise personal movement histories.
- Do not include private user reports, personal addresses, or identifying safety details in public examples.
- Use anonymized or synthetic data for tests, screenshots, demos, and documentation.

## Scope

In scope:

- Authentication, authorization, or moderation bypasses.
- Exposure of private reports, location data, or service credentials.
- Issues that could enable harassment, doxxing, spam, or unsafe public records.

Out of scope:

- General product suggestions.
- Vulnerabilities in third-party services not controlled by this repository.
- Requests for private deployment access.
