# Versioning

This project follows [Semantic Versioning 2.0.0](https://semver.org/spec/v2.0.0.html).

## Release Cadence

There is no fixed release schedule. Releases are made when meaningful changes accumulate or when a feature milestone is reached.

## Version Scheme

Given a version number `MAJOR.MINOR.PATCH`:

- **MAJOR** — Breaking changes to the API, data model, or authentication flow.
- **MINOR** — New features or functionality additions (backward-compatible).
- **PATCH** — Bug fixes, security patches, documentation, or refactoring (backward-compatible).
